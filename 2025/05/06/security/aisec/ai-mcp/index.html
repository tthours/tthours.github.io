ai-mcp

1 简述

Anthropic 发布

Model Context Protocol 模型上下文协议

旨在统一规范大语言模型与外部数据源和工具之间的通信，建立 AI 与外部数据、工具交互的标准，类似中间件

2 架构

核心组件

Host

本地运行的 AI 应用环境，为执行基于 AI 的任务提供环境。

托管 MCP 客户端，确保与外部 MCP 服务器的通信。

Client

MCP 客户端，在主机和 MCP Server 中保持连接，例如：Claude 客户端、Cline等应用程序都内置 MCP Client 客户端。

MCP 客户端通过传输层与 MCP 服务器通信，促进安全、可靠的数据交换以及主机与外部资源之间的顺畅交互。

Server

MCP 插件所对应的服务端。

核心功能

工具

启用外部操作

允许 MCP 服务器调用外部服务和 API ，代表 AI 模型执行操作。

通过上下文自主选择和调用适当的工具，遵循标准化的供应-消费模型，使工具模块化、可重用。

资源

向 AI 模型暴露数据。

资源提供对结构化和非结构化数据集的访问，数据集可以由 MCP 服务器暴露给 AI 模型。

提示

用于工作流程优化的可重用模版。

提示是 MCP 服务器生成和维护的预定义模版和工作流程，用于优化 AI 响应和简化重复性任务。

通信流程

用户 -> Host -> Client -> Server

结构化过程

1、客户端向服务器查询功能的初始请求

2、服务器会初始响应回复，列出可用的工具、资源和提示

3、客户端利用可用的工具、资源和提示

4、建立连接后，持续交换通知。
