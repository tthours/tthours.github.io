7-域3安全架构与工程-PKI和密码应用

7.1 非对称密码

7.1.1 公钥和私钥

公钥密码系统（public key cryptosystem）给每个用户都分配一对密钥：一个公钥和一个私钥。

公钥用于加密，私钥用于解密。

公钥系统适用的密钥必须要比私钥系统所用的密钥更长。

7.1.2 RSA

1977 年 RSA 公钥算法（RSA public key algorithm）。

算法原理：

因式分解大素数乘积

公私钥生成步骤：

1、选择两个大素数（每个都约为 200 位），用 p 和 q 表示

2、计算这两个数的乘积：n = p*q

3、挑选一个满足以下两项要求的数 e

  • e 小于 n
  • e 和 (p-1)(q-1)是互素的–这两个数没有除了 1 以外的公因数

4、找到一个数 d ，使 ed = 1 mod ((p-1)(q-1))

5、把 e 和 n 作为公钥分发给密码系统的所有用户。d 作为私钥保密。

Merkle - Hellman 背包算法

也基于进行因式分解运算。

1984 年被破解。

密钥长度的重要性

数据越关键，保护数据的密钥需要更强。

密钥破解能力随着时间增强，所以数据需要保护的时间越久，保护数据的密钥就需要越强。

7.1.3 EIGamal 算法

EIGamal 算法优势：

算法公开免费，RSA 2000年才公开算法

EIGamal 算法劣势：

加密的任何信息都被加长了一倍，网络传输困难。

7.1.4 椭圆曲线

1985 年提出理念，椭圆曲线密码（elliptic curve cryptography，ECC）投入实际使用。

椭圆曲线定义：

y的平方 = x的3次方 + ax + b

x y a b 都是实数。

每条椭圆曲线都有一个响应的椭圆曲线群（elliptic curve group），由椭圆曲线上的点和位于无限远的点 O 组成。

可以使用一种椭圆曲线加法算法将同一个椭圆曲线群内的两个点 P 和 Q 相加，表达式如下：

P + Q

如果设 Q 是 P 的倍数，可以扩展这道题以引入乘法，表达式如下：

Q = xP

即使 P 和 Q 是已知的，也极难算出 x 。

业界广泛认为，相比于 RSA 以及 Diffie-Hellman 和 ElGamal ，椭圆曲线更难。

7.1.5 Diffie-Hellman 密钥交换

Diffie-Hellman算法是由素数数学原理支撑的。

算法步骤

(1) Richard 和 Sue 商定采用两个大数：一个素数 p 和一个整数 g, 同时设 1<g<p 。

(2) Richard 挑选了一个大随机整数 r 并进行以下计算：

R=g 的 r 次方 mod p

(3) Sue 挑选了一个大随机整数 s 并进行以下计算：

S=g 的 s 次方 mod p

(4) Richard 把 R 发送给 Sue, 而 Sue 把 S 发送给 Richard 。

(5) Richard 随后进行以下计算：

K=S 的 r 次方 mod p

(6) Sue 随后进行以下计算：

K=R 的 s 次方 mod p

Diffie-Hellman 本身不是加密协议，属于密钥交换协议。

常被用于为传输层安全（TLS）的使用创建共享秘密密钥。

Diffie-Hellman 密钥交换算法依赖于大素数的使用。

ECDHE 密钥交换算法是 Diffie-Hellman 方法的变体，用椭圆曲线问题来执行类似的密钥商定过程。

7.1.6 量子密码

量子计算（quantum computing）

量子计算理论：

用名为 “量子比特”（qubit）的多维量子位取代数字计算的二进制 1 和 0 位。

量子霸权（quantum supremacy），将会导致现有的非对称加密算法 RSA、Diffie-Hellman 不再安全。

量子密码（quantum cryptography）系统，创建更新、更复杂的密码算法。

量子密钥分发（quantum key distribution，OKD）执行方案，借助量子计算在两个用户之间创建共享秘密密钥。

对数据的加密需要考虑到量子世界的安全性。

7.2 哈希函数 hash function

数字签名

证明一条消息来自秘密系统中的某一个特定的用户，同时确保消息在双方之间传递的过程中未曾有过改动。

哈希函数目的

提取一条可能会比较长的消息，然后从消息内容中派生一个唯一的输出值，也就是消息摘要（message digest）。

消息摘要

消息摘要由消息的发送者生成，并和整个消息一起发送给接收者。原因如下

  • 接收者用同一个哈希函数根据整条消息重算消息摘要，消息必须完全相同，摘要才可能匹配。
  • 消息摘要可以用来执行数字签名算法。

消息摘要一般为 128 位或更长。

消息摘要越长，完整性检验越可靠。

密码哈希函数基本要求

哈希函数创建一个值，值是唯一的，不可以被逆向推算或“反哈希化运算”

  • 输入可以是任何长度
  • 输出有一个固定长度
  • 哈希函数的计算对任何输入而言都相对容易
  • 哈希函数是单向的
  • 哈希函数抗碰撞，不会有产生相同哈希值的消息

需要知道的哈希算法

安全哈希算法（SHA）

消息摘要5（MD5）

RIPE消息摘要（RIPEMD）

消息身份认证码（HMAC）

可变长度哈希（HAVAL）

7.2.1 SHA

“安全哈希算法”（SHA）及其后继算法 SHA-1、SHA-2 和 SHA-3

SHA-1 已弃用

可以提取任何长度的输入

生成 160 位消息摘要

处理 512 位块消息

如果消息长度不足 512 ，使用附加数据填充消息，确保消息长度为 512 倍数

SHA-2

SHA-1 替代品

SHA-2 变体

  • SHA-256：用 512 位块大小生成 256 位消息摘要
  • SHA-224：用 512 位块大小生成 224 位消息摘要
  • SHA-512：用 1024 位块大小生成 512 位消息摘要
  • SHA-384：用 1024 位块大小生成 384 位消息摘要

SHA-3

Keccak 算法定为 SHA-3 标准

SHA-3 通过一种不同的算法提供与 SHA-2 相同变体和哈希长度

SHA-3 可以提供与 SHA-2 相同的安全强度，但是速度比 SHA-2 慢

可以在硬件中执行，但是使用率不高。

7.2.2 MD5 消息摘要5

MD2 消息摘要2

1989年开发

为 8 位处理器提供安全哈希函数

MD4 消息摘要4

1991年

MD2 的强化版本，支持 32 位处理器，并提高了安全强度。

MD5 消息摘要5

处理 512 位消息块

4 轮不同计算生成一个长度和 MD2 和 MD4 算法相同的摘要（128位）

消息长度必须比 512 位的倍数小 64 位

7.2.3 RIPEMD RIPE消息摘要

RIPE 消息摘要（RIPEMD）系列哈希函数是 SHA 系列的替代方案

包含的函数

  • RIPEMD 生成 128 位摘要，并存在结构化缺陷，不安全
  • RIPEMD-128 取代 RIPEMD，生成 128 位摘要，不安全
  • RIPEMD-160 取代 TIPEMD-128，生成 160 位哈希值，目前认为安全

安全强度来看：
RIPEMD-128 和 RIPEMD-256 相同，不安全

RIPEMD-160 和 RIPEMD-320相同，安全
