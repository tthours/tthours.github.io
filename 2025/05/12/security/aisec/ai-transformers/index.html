transformers

https://github.com/huggingface/transformers/blob/main/i18n/README_zh-hans.md

Hugging Face 出品的 工具包，实现了大量的主流预训练

使用 PyTorch 或 TensorFlow 快速训练一个模型

原理

Transformers 是 Google 提出的一种深度学习架构。

抛弃了传统的循环神经网络（RNN）和卷积神经网络（CNN），完全基于自注意力机制（Self-Attention）

特点

基于自注意力机制（Self-Attention）

支持并行计算（不同于 RNN 的序列号处理，也就是串行处理）

能够捕捉长距离依赖关系

成为现代大模型的基础架构

自注意力机制 Self-Attention

允许模型动态关注输入序列的不同部分。

自注意力机制本身是“排列不变”（Permutation-Invariant）的，即打乱输入序列的顺序，自注意力的计算结果不会改变。

注意力机制让模型在处理每个词时，动态关注输入序列中最相关的其他词。

模型在处理序列时，每个词都会与其他词进行关联计算。

多头注意力（Multi-Head Attention）

单头注意力只能从单个维度（视角）

用多个不同的矩阵，并行计算注意力，捕捉不同类型的关系

残差连接和层归一化

残差连接 Residual Connection

核心作用：解决深层网络训练时“信息丢失”的问题

使用原因：

  • 深层网络的挑战：

    当神经网络层数不断增加时，每一层的微小变化都有可能被逐层放大或衰减，从而导致模型无法训练（例如梯度消失或爆炸）。

    transformer 堆叠多层（编码层和多解码）来增强表达能力，单纯的堆叠会导致原始输入信息在传输过程中逐渐丢失。

  • 解决方案：

    跳过连接 - 在每层的输出中，除了包含输出结果，还包含原始输入，即使某一层变换失效，信息仍能通过“捷径”直接传递到后续层。

层归一化（Layer Normalization, LN）

核心作用：对每一层的输入进行标准化，稳定数值范围，加速训练

使用原因：

  • 内部协变量偏移（Internal Covariate Shift）：

    神经网络每一层的输入分布会随着前一层参数更新而变化（例如均值和方差漂移），导致后续层需要不断适应新的分布，降低训练效率。

  • 解决方案：

    对每一层的单个样本（或单个位置）的所有特征维度进行标准化（均值为0，方差为1），使输入分布保持稳定。

归一化的维度：

  • 层归一化（LN）

    对每个样本（或位置）的所有特征维度归一化（例如对每个单词的512维向量单独标准化）

  • 批归一化（BN）

    对同一批次内所有样本的同一特征维度归一化（例如对所有句子的第1个词的第1维特征统一标准化）

  • Transformer选择LN而非BN，因为序列长度可变（BN需要固定批次内序列长度）

残差连接和层归一化组合效果

在Transformer中，这两者通常成对出现，形成“预激活残差块”（Pre-activation Residual Block）

残差连接：确保信息无损传递，缓解梯度问题

层归一化：稳定数值分布，加速训练

组件 核心作用 关键特点
残差连接 解决深层网络信息丢失问题 跳过连接（Output=x+F(x)），确保梯度顺畅回传。
层归一化 稳定数值分布，加速训练 对每个样本（或位置）的所有特征维度标准化（均值0，方差1），支持变长序列。
组合效果 稳定训练 + 增强表达能力 Transformer中通常以 LayerNorm(x + Sublayer(x)) 的形式出现，形成鲁棒模块。

架构

编码器-解码器（Encoder-Decoder）架构

组件 说明
Encoder（编码器） 对输入序列进行编码，提取高层次特征
Decoder（解码器） 根据编码器的输出和之前生成的 token，生成目标序列

Input → Embedding + Positional Encoding → [Encoder Stack] → [Decoder Stack] → Output

输入表示 Input Embedding + Positional Encoding

Input Embeding 层-输入嵌入

将每个词（token）映射为一个固定维度的向量（转换成计算机能理解的数字向量，也就是一串数字）。

通常使用一个可以学习的嵌入矩阵（Embedding Layer），矩阵大小为[vocab_size,d_model]，vocab_size 是词汇表大小，d_mode 是嵌入维度

关键作用

1、把离散符号变成连续数字：文字 → 数字向量

2、保留语义信息：相似的词，向量也相似

3、压缩信息：用固定长度的向量（如512维）表示一个词，而不是用冗长的one-hot编码（比如词汇表大小=10,000时，one-hot编码是10,000维！）

Positional Encoding

由于 Transformer 没有 RNN 的顺序结构，需要显式加入位置信息，使模型知道词的顺序。

为每个单词的嵌入向量添加一个独特的“位置标记”，使模型能够区分不同位置的单词。

唯一性：每个位置对应唯一的编码

可学习或固定：可以是可学习的参数，或通过固定公式生成（原始论文使用固定公式）

与嵌入维度兼容：编码的维度需与输入嵌入的维度 d_model 一致

编码器 Encoder - 核心结构

每个 Encoder 层包含两个子层：

多头自注意力机制（Multi-Head Self-Attention）

前馈神经网络（Feed-Forward Neural Network，FFN）

多头自注意力机制
  • 每个词都会计算它与序列中其他词的关系（即“关注”其他词的信息）
  • 通过多个“头”（head）并行计算，捕捉不同方面的关系
前馈神经网络 FFN
  • 对每个位置独立进行一个两层的全连接网络变换（通常使用 ReLU 激活函数）

本质：对序列中每个位置的向量进行独立的、相同的非线性变化，增强模型的表达能力

核心作用：

  • 引入非线性，捕捉复杂语义

  • 细化每个词的特征表示（如语法、语义角色）

FFN 和注意力机制

组件 功能 是否跨位置交互 输入输出形状
注意力机制 建模词与词之间的关系 是（跨位置） [batch, seq_len, d_model]
FFN 对每个位置的向量单独非线性变换 否（仅处理当前位置） [batch, seq_len, d_model]

每个子层后都会接一个 残差连接（Residual Connection） 和 层归一化（Layer Normalization）

解码器 Decoder

每个 Decoder 层包含三个子层：

掩码多头自注意力机制（Masked Multi-Head Self-Attention）

Encoder-Decoder 注意力机制（Encoder-Decoder Attention）

前馈神经网络（Feed-Forward Neural Network，FFN）

掩码多头自注意力机制
  • 类似于 Encoder 的自注意力，但增加了掩码（Masking），防止解码器看到未来的词（保证自回归生成）
Encoder-Decoder 注意力机制
  • 解码器的 Query 来自解码器自身，Key 和 Value 来自编码器的输出
  • 使解码器能够“关注”编码器输出的相关信息
前馈神经网络 FFN
  • 与 Encoder 中的 FFN 相同，对每个位置独立进行变换

每个子层后都会接一个 残差连接（Residual Connection） 和 层归一化（Layer Normalization）

输出层（Output Layer）

  • Decoder 的最后一层输出经过一个线性层（全连接层）映射到词汇表大小的向量。
  • 然后通过 Softmax 函数得到每个词的概率分布，用于预测下一个词

AutoClass

AutoModelForSequenceClassification 和 AutoTokenizer 一起支持在上面用到的 pipeline()

AutoClass 是一个能够通过预训练模型的名称或路径自动查找其架构的快捷方式

AutoTokenizer

分词器负责预处理文本，将文本转换为用于输入模型的数字数组

tokenizer
语法

max_length：模型支持的最大序列长度，超过后会被截断

padding： True

自动对批次中的文本进行填充（Padding），使所有序列长度一致

填充规则：

  • 按照 padding_side ，默认为 right，在右侧填充
  • 填充符号由 pad_token 决定

truncation：True

自动截断超过 max_length 的序列

截断规则：

  • 按照 truncation_side ，默认为 right ，从右侧截断
  • 截断后保留的 token 数量不超过 max_length

return_tensors

指定返回张量的框架类

可选值：

  • pt：返回 PyTorch 张量（torch.Tensor）

    返回值中包含 input_ids、attention_mask 字段

    input_ids：每个 token 的 ID（包括特殊 token 如 [CLS] 和 [SEP]）

    attention_mask：标记哪些位置是有效 token（1），哪些是填充（0）

  • tf：返回 TensorFlow 张量（tf.Tensor）

  • np：返回 NumPy 数组（np.ndarray）
