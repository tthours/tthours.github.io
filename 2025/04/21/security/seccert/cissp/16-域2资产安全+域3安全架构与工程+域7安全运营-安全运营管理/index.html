16.1 应用基本的安全运营概念

安全运营实践的主要目的是保护资产。

安全运营实践有助于识别威胁和漏洞，并实施控制措施来降低资产的风险。

应尽关心（due care）和尽职审查（due diligence）是指采取合理的措施，持续保护组织的资产。

高管对应尽关心和尽职审查负有直接责任。

16.1.1 因需可知最小特权

1. 因需可知的访问 need to know

因需可知原则要求仅授予用户执行工作所需数据或资源的访问权限。

因需可知原则主要目的是让秘密信息保持秘密状态。

因需可知通常和安全许可（security clearance）关联。

2. 最小特权原则

最小特权原则规定，主体仅被授予执行工作所需的特权。

特权既包括访问数据的权限也包括执行系统任务的权利。

对于数据，特权是指控制写入、创建、更改或删除数据的能力。

最小特权原则保护数据的完整性和保密性，或者保护其他文件的完整性。

最小特权原则不仅适用于访问数据，还适用于访问系统。

违反最小特权原则的情况：

将所有用户添加到本地管理员组或授予计算机的 root 权限。

最小特权原则通常侧重于保护用户特权受到限制，但也适用于应用程序或进程，确保都是最小特权

16.1.2 职责分离和责任

职责分离SoD 和责任确保个体无法完全控制关键功能或系统。

确保没有任何一个人可危及系统或系统安全。

职责分离策略形成一个制衡系统，其中两个或多个用户验证彼此的行为，并且必须协同完成必要的工作任务。

职责分离策略有助于减少欺诈行为。

组织经常把流程分解为多个任务或职责，并将这些职责分配给不同的人来预防欺诈。

执行职责分离的另一种方式是多个受信任个体分担安全或管理的功能及职能。

16.1.3 双人控制

双人控制（two- person control）双人制。

要求经过两个人批准后才能执行关键任务。

实现同行评审并减少串通和欺诈的可能性。

可以讲一些特权活动配置成双人控制。

知识分割（split knowledge）基本思想是将执行操作所需的信息或特权分配给两个或多个用户。

16.1.4 岗位轮换

岗位轮换（职责轮换）是指员工进行岗位轮换或和其他员工进行职责轮换。

可以实现同行评审、减少欺诈行为并实现交叉培训。

交叉培训可以减少环境对任何个体的依赖。

岗位轮换可以充当威慑和检测机制。

16.1.5 强制休假

要求员工强制休假一周或两周。

提供一种同行评审形式，有助于发现欺诈和串通行为。

强制休假也可以充当威慑和检测机制。

16.1.6 特权账户管理PAM

特权账户管理（PAM）解决方案限制特权账户的访问权限，或者检测账户是否使用了提升的特权。

Microsoft 域中：

特权账户包括本地管理员账户（完全控制计算机）、域管理员组中的用户（完全控制域内任何计算机）和企业管理员组的用户（完全控制域林中所有域）。

Linux 中：

特权账户包括使用 root 账户或通过 sudo 命令授予 root 访问权限的用户。

微软域（domain）包含限制特权访问的 PAM解决方案。

微软域基于即时管理原则。用户被分配到特权组，但是没有特权，需要的时候申请使用，授权存在有效期。

特权账户管理会监控特权账户执行的操作（新建账户、添加新路由、更改防火墙配置以及访问系统日志和审计文件）

监控特权

需要和其他基本原则相组合。

阻止员工滥用特权，并在受信任员工滥用特权时检测到。

自动化工具可以监控特殊权限的使用情况。

检测高级持续性威胁（APT）

通过监控提升特权的使用，还可以检测高级持续性威胁（advanced persistent threat，APT）活动。

常见特权操作：

  • 访问和删除日志
  • 创建并操作账户
  • 控制通信路径（例如：开启 3389 或关闭主机防火墙）
  • 运行各种脚本
  • 创建和安排计划任务

16.1.7 服务水平协议 SLA

服务水平协议（service level agreement，SLA）是组织和外部实体之间的协议。

SLA 规定了绩效预期，并常包括针对未实现预期的惩罚条款。

谅解备忘录（MOU）：记录两个实体合作达成共同目标的意愿。

MOU 不够正式，缺乏处罚条款。

16.2 解决人员安全和安保问题

人员安全是安全运营的一个基本要素。

人是无法替代的。

16.2.1 胁迫 duress

员工单独工作时，胁迫（duress）系统非常有用。

简单胁迫系统只有一个发送遇险呼叫的按钮，通知监控人员。

安全系统通常包括口令，可以使用不同的口令来确定情况是正常的还是存在问题的。

16.2.2 出差

电子设备的安全风险

  • 敏感数据

    最好出差设备不存储任何敏感信息。

    如果需要敏感信息，使用强加密手段保护。

  • 恶意软件和监控设备

    始终保持对设备的实际控制。

    出差使用临时设备，结束后清除设备数据，重装系统。

  • 免费 Wi-Fi

    用户应该使用自己创建的连接互联网的方法

    例如：无线热点设备

  • VPN

    访问虚拟专用网络（VPN）来创建安全连接。

16.2.3 应急管理

应急管理计划及实践帮助组织在灾难发生后解决人员安全和安保问题。

不管面临何种灾难，都应该首先考虑人员安全。

16.2.4 安全培训和意识

适当的培训和意识计划，有助于确保员工了解胁迫系统、出差最佳实践、应急管理计划以及人员安全的最佳实践。

16.3 安全配置资源

16.3.1 信息和资产所有权

数据所有者对数据担负最终组织责任。

高级管理人员对其他资产（如硬件资产）负有最终责任。

16.3.2 资产管理

资产管理是指管理有形资产和无形资产。

资产管理通常从资产清单开始，要求跟踪资产，采取额外措施在整个生命周期内保护资产。

有形资产：硬件资产和软件资产

无形资产：专利、版权、公司声誉和其他代表潜在收入的资产。

1. 硬件资产清单

自动配置管理系统（configuration management system，CMS）管理硬件资产。

CMS 目的是配置管理，需要连接到硬件系统。

硬件资产包括：计算机、服务器、路由器、交换机和外围设备等 IT 资源。

使用数据库和库存应用程序在整个设备生命周期内执行资产盘点和跟踪硬件资产。

常见方法：

条形码

射频识别（radio frequency identification，RFID）标签

RFID 更昂贵，可以明显减少资产盘点所需的时间。

净化（sanitize）

净化设备的操作会删除设备上的所有数据，确保未授权的人员无法访问敏感信息。

需要使用检查表 checklist 清理系统。

检查表包含对系统内各类介质的净化步骤，净化对象包括系统内置硬盘、非易失性存储器，以及 CD、DVD 和 USB 闪存驱动器等可移动介质

保存敏感数据的便携式介质也可以作为资产进行管理。

2. 软件资产清单

软件资产包括操作系统和应用程序。

使用许可密钥（license key）来激活软件。

激活往往需要连接互联网许可服务器，以防止盗版行为。

应该监测软件的许可合规来规避法律纠纷。

软件许可可以确保系统没有按照未经授权的软件。

3. 无形资产清单

需要跟踪无形资产并加以保护。

高级管理团队通常是无形资产的所有者。

评估资产为组织带来的收益来确定无形资产的价值。

公认会计原则（GAAP）在其资产负债表上报告无形资产的价值。

组织至少每年审查一次无形资产。

16.4 实施资源保护

组织应用各种资源保护技术来确保在整个生命周期内安全配置和管理资源。

16.4.1 媒介管理

媒介管理是指为保护媒介及其存储数据而采取的步骤。

媒介泛指任何可存储数据的设备。

智能手机也属于媒介。

媒介管理不仅涵盖备份磁带，还包括任何存储数据的介质类型，还包括任何类型的硬拷贝数据。

16.4.2 媒介保护技术

存储在严格访问控制的安全场所，安全场所需要具备温度和湿度的控制措施。

媒介管理包括技术控制措施，以限制计算机系统对媒介的访问，例如：禁止使用 USB

正确的媒介管理可以直接解决保密性、完整性和可用性问题。

正确标记、处理和保管媒介时，可以防止未经授权的泄露（保密性破坏）、未经授权的修改（完整性破坏）和未经授权的破坏（可用性破坏）。

1. 磁带媒介

数据备份存储在磁盘上。

至少保留两份数据备份，一份保留在本地，一份保存在异地。

管理磁带媒介的实用指南：

  • 如果不需要使用，应该将新媒介保管在原始密封包装中。
  • 打开媒介包装时，务必小心，不要损坏介质，包括避免碰到尖锐物体，不扭曲或弯曲磁带媒介。
  • 避免把磁带媒介暴露在极端温度下；不要靠近加热器、散热器、空调或其他极端温度源。
  • 请勿使用损坏的、暴露在超标尘垢中或摔过的媒介。
  • 应该使媒介免受外界环境的影响；避开阳光、潮气、高温和低温。媒介使用前应该放置 24 小时来适应环境。
  • 从出发点到安全的场外存储设施，应该保证媒介的适当安全。在运输过程中，介质容易遭受损坏和盗窃。
  • 应该根据介质存储数据的分类等级，保证媒介在整个生命周期内的安全。
  • 应该考虑对备份数据加密，防止在备份磁带丢失或被盗时造成数据泄露。

2. 移动设备

移动设备包括智能手机和平板电脑，移动设备配置了内置存储或移动存储卡。

带有附件、联系人和日程信息的电子邮件也是数据。

谨记移动设备拥有数据存储功能，如果移动设备存储敏感数据，务必采取措施保护。

3. 媒介管理生命周期

所有媒介都拥有一个有用但有限的生命周期。

可重复使用的媒介会受到平均故障时间（mean time to failure，MTTF）的影响。

许多因素会影响媒介使用寿命，并可能降低预期指标。

务必监控备份是否出现错误，如果开始出现错误，就应该替换。

备份媒介一旦达到平均故障时间，就应该被销毁。

磁带存储数据的密级决定媒介销毁的方法。

高密级磁带进行消磁，然后存放进行销毁（散装粉碎机或焚烧炉中进行销毁）

消磁操作不会删除 SSD 存储的数据，内置的擦除命令也不会清理整个硬盘。

不要删除数据，应该销毁 SSD 。

MTTF（平均故障时间） 的计算通常针对出现故障时无法修复的项目。

MTBF（平均故障间隔时间）的计算是项目两个可修复故障之间的时间间隔。

16.5 云托管服务

基于云的资产包括一个组织使用云计算访问的任何资源。

云计算是指可以按需访问的计算资源，云计算资源具有高可用性和易扩展性。

租用和本地部署优缺点：
租用优点可以提供便利，但是不受组织的直接控制。

本地部署可以有更大的控制权。

在云上存储数据时，必须确保安全控制到位。

所有敏感数据都应该加密，包括发送到云上的传输数据和存储在云上的静态数据。

云租户应该对加密进行管理，包括掌控所有加密密钥，而不是使用云服务商掌握的加密措施。

使用加密擦除方法来销毁数据。

加密擦除方法永久删除加密密钥。

16.5.1 使用云服务模型分担责任

根据云服务模式不同，资产维护和安全责任程度也不相同，包括维护资产，确保资产功能，以及使用系统和应用程序与当前补丁保持同步。

软件即服务（software as a service ,SaaS）

提供通常可通过 Web 浏览器访问的全功能应用程序。

用户不需要管理或控制任何云资产。

平台即服务（platform as a service，PaaS）

为用户提供一个计算平台，包括硬件、操作系统和运行时环境。

云服务提供商负责维护主机和底层云基础架构。

基础架构即服务（infrastructure as a service，IaaS）

为用户提供基本的计算资源。

资源包括服务器、存储和网络资源。

操作系统、应用程序需要用户部署维护。

4 种云部署模型

  • 公有云模型：

    可供任何用户租用或租赁的资产，资产由外部云服务提供商（CSP）托管。

    服务水平协议可以有效地确保 CSP 服务水平。

  • 私有云模型：

    适用于单个组织的云资产。

    组织可以使用自有资源创建和托管私有云。

    组织负责所有维护工作。

    也可以从第三方租用资源，维护要求根据服务模型（SaaS、PaaS 或 IaaS）进行拆分。

  • 社区云模型：

    为两个或多个共同关切的组织（例如有类似的任务、安全要求、策略）提供云资产。

    云资产由一个或多个组织拥有和管理。

    维护责任根据资产托管方和服务模型进行分配。

  • 混合云模型：

    两个或多个云的组合

    混合云由一种提供数据和程序迁移的技术结合在一起。

    与社区云模型类似。

    维护责任根据资产托管方和服务模型进行分配。

16.5.2 可扩展性和弹性

可扩展性是指系统通过扩充附加资源来处理超过载荷的能力。

弹性是指系统根据负载的增加和减少来动态添加或削减资源的能力。

关键点：

弹性方法不需要通过关闭系统来添加资源，资源可以自动添加或删除。

可扩展性方法不是自动或动态的，需要手动干预。

16.10 考试要点

了解因需可知和最小特权原则之间的区别

因需可知和最小特权原则是安全网络环境遵循的两个标准 IT 安全原则。

因需可知和最小特权原则限制人员对数据和系统的访问，以便用户和其他主体只能访问需要的内容。

这种受限的访问有助于预防安全事件，且有助于在事件发生时限制事件的影响范围。

如果组织不遵循这些原则，安全事件将对组织造成更大的损害。

理解职责分离和岗位轮换

职责分离是一项基本安全原则，确保单个的人无法掌握关键职能或关键的系统要素。

通过岗位轮换，员工可以轮换到不同的工作岗位，或者任务可以分配给不同员工。

串通是指多人协同一些未经授权的或非法的行为。

在没有出现串通的情况下，这些策略可以通过限制个人行为来预防欺诈。

理解监控特权操作的重要性

虽然特权用户是受信任的，但他们可能滥用其特权。因此，有必要监控所有特权的分配及使用。

监控特权操作的目的是确保受信任的员工不会滥用被授予的特权。

因为攻击者通常在攻击时使用特权，监视特权操作还可以检测到一些攻击。

高级特权管理可限制用户拥有高级特权的时间。

理解服务水平协议

组织与供应商等外部实体签订服务水平协议。

SLA 规定了性能期望，例如最长停机时间。

SLA 通常会包含处罚条款，以防供应商达不到预期。

关注人员安全

胁迫系统可使警卫在紧急情况下发出警报，应急管理计划可帮助组织应对灾难。

当员工出差时，员工需要意识到风险，特别是当出差到不同的国家时。

‘

安全配置和意识计划可确保员工了解这些风险以及减轻风险的方法。

理解安全配置概念

资源的安全配置包括确保资源以安全方式部署并在其整个生命周期中以安全方式维护。

资产管理跟踪有形资产（硬件和软件）和无形资产（如专利、商标、公司商誉和版权）。

了解如何管理和保护媒介

媒介管理技术追踪保存敏感数据的媒介。

媒介在其整个生命周期都受到保护，并在不再需要时销毁。

理解 SaaS、PaaS 和 IaaS 之间的区别

软件即服务（SaaS）模型提供通常可通过 Web 浏览器访问的全功能应用程序。

平台即服务（PaaS）模型为用户提供计算平台，包括硬件、操作系统和运行时环境。

基础架构即服务（IaaS）模型提供基本的计算资源，例如服务器、存储和网络资源。

识别云托管服务的安全问题

云托管服务包括云存储或通过云访问的任何资源。

将数据存储在云中会增加风险，因此可能需要额外的措施来保护数据，具体措施取决于数据的价值。

在租赁基于云的服务时，必须了解谁负责维护和安全。

云服务提供商在 IaaS 模型中提供最少的维护和安全。

解读配置和变更管理控制

通过有效的配置和变更管理计划，可以预防业务中断及许多其他的事件。

配置管理确保系统采用相近配置，并且系统配置是可知悉和可记录的。

基线确保所部署的系统有相同基线或相同启动点，而镜像是一种常规基线技术。

变更管理有助于预防未授权的变更，进而减少业务中断并防止削弱安全。

变更管理流程规定了变更的请求、批准、测试和记录。

版本控制使用标签或编号系统来跟踪软件版本的变更。

理解补丁管理

补丁管理可以确保系统和当前补丁保持同步。

应该认识到有效管理补丁计划包括补丁评估、测试、批准和部署。

此外，系统审核将验证已批准的补丁是否已部署到系统。

补丁管理通常与变更和配置管理结合在一起，确保文档内容可以显示变更。

当缺少有效补丁管理计划时，组织往往会遇到因已知问题导致的中断及事件，而这些问题原本是能预防的。

解读漏洞管理

漏洞管理包括例行漏洞扫描和定期漏洞评估。

漏洞扫描器可以检测已知的安全漏洞和脆弱点，如未打补丁或使用了弱密码。

漏洞管理可以生成报告，报告可以指出系统存在的漏洞，并对补丁管理计划进行有效检查。

漏洞评估不仅涉及技术扫描，还包括漏洞检测的审查和审核。
