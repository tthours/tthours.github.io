17-域7安全运营+域8软件开发安全-事件的预防和响应

有效的事件管理可以帮助组织在攻击发生时做出响应，以限制攻击的范围。

组织可以通过预防措施抵御和检测攻击。

日志记录和监控可确保安全控制措施到位，并提供所需的保护。

17.1 实施事件管理

任何安全方案都以预防安全事件作为主要目标之一。

事件管理的主要目标是将组织受到的影响降至最低。

17.1.1 事件的定义

事件一般指会对组织资产的保密性、完整性或可用性产生负面影响的任何事情。包含各种事件，自然灾害、意外事件等。

计算机安全事件（安全事件）通常是指由攻击导致的事件，或由用户的恶意或蓄意行为导致的事件。

常见安全事件例子：

  • 任何未遂的网络入侵
  • 任何未遂的拒绝服务攻击
  • 对恶意软件的任何检测
  • 对数据的任何未经授权的访问
  • 对安全策略的任何违背行为

17.1.1 事件管理步骤

事件管理是一种持续开展的活动，“总结教训”阶段的结果将用来改进检测方法或帮助防止事件在未来反复发生。

检测 -> 响应 -> 抑制 -> 报告 -> 恢复 -> 补救 -> 总结教训

事件管理不包含对攻击者的反击，反击会适得其反。

反击存在违法、打击无辜者的风险。

1. 检测

检测潜在事件的方法：

  • 入侵检测和预防系统
  • 反恶意软件在检测到恶意软件时会弹出警告窗口
  • 自动化工具定期扫描审计日志，以找出预先定义好的事件。检测到后会向管理员发出警报。
  • 最终用户会检测到非常规活动并对技术人员或管理员求助

IT 人员需要对检测到的事件进行调查判断是否真实。

IT 人员需要经过专门培训区分问题和安全事件。

确认为安全事件后，将进入下一步骤响应。

2. 响应

响应因事件严重程度而异。

专门的事件响应团队，处理重大安全事件。
计算机事件响应团队 CIRT

计算机安全事件响应团队 CSIRT

组织对事件响应的速度越快，把损害控制在有限范围的机会就越大。

务必在调查过程中保护好可以充当证据的所有数据。

计算机涉及安全事件时不可关闭，关闭会导致 RAM 中的数据丢失。

3. 抑制

抑制措施目的在于遏制事件的发展。

有效事件管理的主要目的之一是限制事件的影响和范围。

可以在不让攻击者知道攻击已被发现的情况下采取措施抑制事件。

监控攻击者的行为并确定攻击范围。

4. 报告

报告是指在组织内通报事件并将情况上报给组织外的相关部门和个人。

高管需要对严重安全破坏事件知情。

每个组织都有责任了解哪些法律适用于自身，并遵守这些法律。

如果对系统的一次攻击导致攻击者掌握了有关个人身份信息，系统负责人有责任把攻击及攻击者访问了哪些数据的情况告知用户。

组织在响应严重安全事件的过程中应该考虑把事件报告给执法部门。

如果个人信息被暴露，组织必须向执法部门报告安全事件。

5. 恢复

恢复系统或使其返回正常运行状态。

小事件，重启系统。

重大事件，重建系统，从最近的备份中恢复所有数据。

重建系统时，必须确保系统配置恰当，安全性至少要达到事件之前的水平。

系统配置检查事项：

  • 访问控制列表（包括防火墙或路由器规则）
  • 服务和协议（确保禁用或移除不需要的服务和协议）
  • 补丁（确保打上了所有最新补丁）
  • 用户账号（确保账号更改了默认配置）
  • 破坏（确保遭到破坏的所有地方都得到了修复）

6. 补救

在补救阶段，安全人员首先查看事件，查明事件发生原因，采取措施防止事件再次发生。

需要进行一次根本原因分析。

根本原因分析通过剖析事件来判断事件起因。

缺乏最新补丁，补救措施为执行补丁管理方案。

缺乏输入验证技术，补救措施为风险应用程序，增加输入验证功能。

7. 总结教训

总结教训阶段，安全人员查验事件发生和响应的整个过程，以确定是否要总结经验教训。

安全人员在查验事件响应时寻求响应行动中需要改进的方面。

总结教训阶段结果反馈给事件管理的检测环节。

事件响应团队总结经验教训后，编写报告。根据结果建议改进程序，增加安全控制，甚至更改策略。

管理层决定采纳哪些建议，并对没有采纳建议而存在的风险负责。

17.2 实施检测和预防措施

无论预防措施多么有效，事件依然会发生。

预防性控制

预防性控制（preventative control）力求阻碍或组织不必要或未经授权的活动发生。

预防性控制示例：

围栏

锁具

生物特征识别

职责分离策略

岗位轮换策略

数据分类

访问控制方法

加密

智能卡

回呼程序

安全策略

安全意识培训

杀毒软件

防火墙

入侵预防系统

检测性控制

检测性控制（detective control）力求发现或检测不必要的或未经授权的活动。

检测性控制在事件发生后运行，只有在事件发生后才能发现。

检测性控制的例子：

保安人员

运动检测器

对闭路电视监控系统捕捉的事件进行记录和审查

岗位轮换策略

强制休假策略

审计踪迹

蜜罐或蜜网

入侵检测系统

违规报告

对用户的监督和审查

事件调查

17.2.1 基本预防措施

保持系统和应用程序即时更新

移除或禁用不需要的服务和协议

使用入侵检测和预防系统

使用最新版反恶意软件程序

使用防火墙

执行配置和系统管理流程

必须时刻保持警惕，打上最新补丁并保持适当配置。

防火墙和入侵检测预防系统检测和收集证据。

17.2.2 了解攻击

了解常见的攻击方法，以便于采取措施预防攻击，识别攻击做出适当的响应。

1. 僵尸网

多个僵尸在一个网络中形成一个僵尸网。

僵尸网中的计算机依照攻击者的指令做任何事情。

僵尸牧人通过一台或多台命令和控制（command-and-control，C&C 或 C2）服务器操控僵尸网中的所有计算机。

僵尸会被动或主动联系服务器。

僵尸危害：
系统中安装恶意软件

搜索口令或感兴趣的文件

捕捉用户击键动作

发动攻击

防止拉入僵尸网络的防御方法：深度防御战略，实施多层次的安全保护策略

最新反恶意软件程序+最新安全补丁。

对用户开展教育，类似防范钓鱼邮件。

保持浏览器及其插件即时更新。

不要禁用浏览器内部安全性能，例如沙箱。

2. 拒绝服务攻击 DoS

拒绝服务（denial-of-service，DoS）攻击阻止系统处理或响应对资源和对象的合法访问或请求。

DoS 攻击只是让系统变慢，不会阻止系统响应任何合法通信流。

常见 DoS 攻击形式：

  • 发送大量数据包，导致服务器瘫痪
  • 利用操作系统、服务或应用程序的已知缺陷或漏洞，利用缺陷导致系统崩溃或 CPU 被完全占用

DoS 攻击可导致性能下降、系统崩溃、系统重启、数据损毁、服务被拦截等。

DoS 攻击中的源地址很少是攻击者的 IP 地址。

DoS 攻击可以来自单个系统，也可以以单个系统为目标。

分布式拒绝服务（distributed denial-of-service，DDoS）攻击：多个系统同时攻击一个系统。

攻击者常用僵尸网发动攻击。

DDoS 工具针对面向互联网的系统。

预防措施和抑制DDoS 服务可以防止和抑制攻击。

分布式反射型拒绝服务（distributed reflective denial-of-service，DRDoS）攻击是一种 DoS 变体，将反射方法用于攻击。

DNS 中毒攻击、smurf 攻击和 fraggle 攻击 就是 DRDoS。

3. SYN 洪水攻击

SYN 洪水攻击（SYN flood attack）是一种常见的 DoS 攻击形式。

中断被传输控制协议（TCP）用来发起通信会话的标准三次握手。

攻击者发出多个 SYN 包，绝不用 ACK 包完成连接。

不完整会话消耗资源。

攻击消耗可用内存和处理能力，致使受害者系统变慢或崩溃。

攻击者每个 SYN 攻击包伪造源地址。

SYN 洪水攻击防御方法：

利用 SYN cookie，检查 SYN cookie 并建立会话。

缩短服务器等待 ACK 的时间，默认是 3 分钟，缩短时间释放内存。

TCP 复位攻击

TCP 复位攻击是操纵 TCP 会话的另一种攻击手段。

攻击者在 RST 包中伪造源 IP 地址并切断活跃会话，系统需要重新建联。

威胁需要通过持久会话和其他系统交换数据的系统。

4. Smurf 和 fraggle 攻击-UDP

Smurf 攻击是洪水攻击的一种类型。

使用的是 ICMP 回声包。

攻击者以广播形式把 ICMP 请求数据包发送给网络上所有系统并伪造源 IP 地址，所有系统会把响应包发送给被伪造的 IP 地址，从而导致受害者被淹没。

Smurf 攻击通过路由器发送定向广播，形成一个放大网（也叫 Smurf 放大器）。

Smurf 攻击防御方法

1、正确配置路由器，不再转发定向广播通信流

2、防火墙、路由器，甚至服务器上禁用 ICMP

fraggle 攻击利用的是 UDP 7 端口和 19 端口。

fraggle 攻击利用假受害者 IP 地址广播 UDP 数据包，造成网络上所有系统都向受害者发送通信流。

5. ping 洪水

ping 洪水攻击用 ping 请求淹没受害者。

利用僵尸网内的僵尸发起攻击，系统会在响应 ping 请求的过程中不堪重负。

ping 洪水攻击防御方法

拦截 ICMP 请求包

入侵检测系统可以检测出 ping 洪水，然后修改环境参数，拦截 ICMP 回声请求

死亡之 ping

使用超大 ping 数据包，导致超出系统的处理能力，导致系统崩溃

泪滴

把数据包打碎，使系统无法重新组成原始状态，导致系统崩溃

land 攻击

攻击者把 受害者的 IP 地址同时作为 源 IP 地址和目标 IP 地址，向受害者发送假 SYN 包。

抵御任何缓冲区溢出攻击的最佳保护措施之一是给系统更新补丁。

6. 零日利用

零日利用（zero-day exploit）是指利用别人还不知道漏洞的攻击。

不同情境下零日利用的词义：

攻击者最先发现漏洞：零日攻击的普通定义。

供应商获悉漏洞但还没有发布补丁：补丁未出现的时间段利用漏洞被称为零日利用。

供应商发布补丁，系统在 24 小时内遭到攻击：说法并不普遍。

零日攻击利用防御方法

确保系统不运行不需要的服务和协议，以缩小系统的受攻击面

启动基于网络和基于主机的防火墙，以限制潜在的恶意通信流

用入侵检测和预防系统来帮助检测和拦截潜在攻击

使用蜜罐观察攻击情况，揭示使用零日利用的攻击

7. 中间人攻击

在通信的两个端点之间建立位置，意味着发生了中间人（man-in-the-middle，MITM）攻击（有时候被称为路径攻击）

中间人攻击分两种类型：
一种是复制或嗅探两个通信参与方之间的通信流。

另一种是攻击者把自己定位在通信线路上，充当通信的存储转发或代理机制。

中间人攻击需要多重攻击配合，在客户端面前伪造为服务端，在服务端面前伪造为客户端。

中间人攻击防御方法

使用虚拟专用网（VPN）来规避攻击

系统即时更新补丁可以组织一些

入侵检测系统无法检测出来，但是能检测到通信链路上发生的异常活动并告警。

8. 蓄意破坏

员工蓄意破坏（sabotage）是指员工对其组织实施破坏的一种犯罪行为。

蓄意破坏的防御方法

在离职后禁用员工账号的访问权

加大审计力度、监测异常或未经授权的活动

确保员工与管理层沟通顺畅，适当补偿员工

17.2.3 入侵检测和预防系统

入侵检测系统（IDS）和入侵预防系统（IPS）是常被组织用来检测和预防攻击的两种方法。

当攻击者绕过或挫败安全机制并访问组织资源时，就意味着发生了入侵（intrusion）。

入侵检测（intrusion detection）是一种特殊的监测形式，监控所发生的事件（往往是实时的），以检测表明出现潜在事件或入侵的异常活动。

入侵检测系统（intrusion detection system，IDS）自动检查日志和实时系统事件，以检测入侵企图和系统故障。

由于入侵预防系统（IPS）包含检测能力，因此也称为入侵检测和预防系统（IDPS）

IDS 的主要目的是提供一种可对入侵做出及时和准确响应的方法。

IDS 可用于检测外部 DoS / DDoS 攻击和内部传播类似蠕虫的攻击。

入侵预防系统 IPS 包含 IDS 的所有功能，也可以采取额外措施来阻止或预防入侵。

1. 基于知识检测和基于行为检测

IDS 通过监测网络通信流和检查日志来主动监视可疑活动。

IDS 从路由器、防火墙等传感器获取日志分析。

IDS 评估数据的两种方法：

基于知识检测：使用与反恶意软件程序所用签名定义类似的签名

基于行为检测：不使用签名，而是将活动与正常表现基线进行对比，从而检测出异常行为。

基于知识检测

基于知识检测（knowledge-based detection）也被称为基于签名检测（signature-based detection）或模式匹配检测，是最常用的检测方法。

使用 IDS 供应商开发的已知攻击数据库。

优势：假阳性率很低

劣势：只对已知的攻击方法有效，新攻击或改动的攻击方式识别不出来

IDS 数据库必须定期用新攻击签名更新（提供自动更新签名的方法）

基于行为检测

基于行为检测（behavior-based detection），也被称为统计入侵检测、异常检测或启发式检测。

首先创建正常活动和事件的基线，用基线来确定正常活动。

基线通常在一个有限时段内创建。

网络发生变化，基线也需要更新。

用户和实体行为分析（UEBA ）功能，根据个人行为创建用户文档（类似于网络基线）

异常分析提升了 IDS 的能力，识别和响应流量或活动的突然增加、多次失败登录尝试、非正常工作意外的活动。

基于行为的 IDS 可以被认为是专家系统或伪人工智能系统的标签。

基于行为的 IDS 可以学习事件并对事件做出假设。

基于行为的 IDS 可以检测出没有签名、尚不能被基于签名方法检测到的较新攻击

事件层面概念：

真阳性：有事件发生并被检测出来。

假阴性：有事件发生但没有被检测出来。

假阳性：有事件被检测出来但实际没有发生。

真阴性：没有发生事件而且没有检测出事件。

生物特征识别概念

真阳性：注册用户尝试验证身份并通过验证

假阴性：注册用户尝试验证身份但没有通过验证（或被拒绝）

假阳性：冒名顶替者尝试验证身份并通过验证

真阴性：冒名顶替者尝试验证身份但没有通过验证。

基于行为的 IDS 缺点：

经常发出假警报，也叫假阳性。

基于签名的 IDS 假警报率（假阳性）很低，但是有很高的假阴性率，尤其是面对新攻击。

在允许 IDC 发出大量假警报与确保 IDS 报告真实攻击之间找到平衡点。

2. IDS 响应

IDS 检测到事件后触发警报或发出警示，会以被动或主动的方式做出响应。

主动响应

记录事件和发出通知，更改环境以阻止活动。

修改 ACL 以拦截基于端口、协议和源地址的通信流。

还可以拦截可疑或行为不良用户对资源的访问。

被动响应

记录事件和发出通知。

被动响应以电子邮件或短信等不同方式发送给管理员。

IPS 都是被安放在承载通信流的线路上的。

3. 基于主机的 IDS 和 基于网络的 IDS

IDS 通常分为基于 主机和基于网络的。

基于主机的 IDS （host-based IDS，HIDS）监测一台计算机或主机。

基于网络的 IDS （network-based IDS ，NIDS）通过观察网络通信流模式监测网络。

基于应用的 IDS ，是基于网络 IDS 的一种特定类型，监测两个或多个服务器之间特定应用程序的通信流。

基于主机的 IDS-HIDS

监测计算机的活动，包括进程调用以及系统、应用程序、安全措施和基于主机防火墙日志记录的信息。

优势：

检查事件的详细程度超过 NIDS

能精确定位被攻击破坏的文件

可以跟踪攻击者所用的进程

HIDS 能检测出 NIDS 检测出来的主机系统异常情况

主机IDS 包含反恶意软件能力

劣势：

HIDS 管理成本更高

HIDS 不能检测其他系统受到的网络攻击

HIDS 需要消耗大量系统资源，降低主机系统的性能

HIDS 更容易被入侵者发现和禁用

只在关键服务器上安装 HIDS

基于网络的 IDS-NIDS

NIDS 监测并评估网络活动，以找出攻击或异常事件。

NIDS 可以通过远程传感器监测大型网络，使用传感器在关键网络位置收集数据，将数据发送给中央管理控制台，例如：安全信息和事件管理（SIEM）系统。

监测加密通信流

使用 TLS 解密器，也叫 SSL 解密器。

TLS 解密器检测 TLS 通信流，采取措施对其进行解密，然后把解密的通信流发送给 IDPS 进行检查。

TLS 解密器检测和拦截内部客户端和互联网服务器之间的 TLS 握手，建立两个 HTTPS 会话。

TLS 解密器弱点：

高级持续性威胁（APT）在将通信流从网络中过滤出来前就已经进行加密，由于加密是在客户端上执行的，而不是在 TLS 会话中加密。

交换机端口被用于映射所有流量，也被称为交换端口分析器（SPAN）端口

NIDS 的中央控制台安装在经过加固的专用计算机上，减少了 NIDS 的漏洞，可以让 NIDS 在几乎不可见的状态下运行。

单独部署不会对其他设备性能产生影响，可以增加 NIDS 来平衡负载。

NIDS 通过执行反向地址解析协议（RARP）或反向域名系统（DNS）查找发现攻击源。

NIDS 能够检测到刚刚发起或正在进行的攻击。

不要对入侵者发起反击或尝试反向黑客攻击，不道德、风险也很大，也是非法的，记录日志和嗅探获取证据。

4. 入侵预防系统 IPS

入侵预防系统（intrusion prevention system，IPS）是一种特殊类型的主动响应 IDS，可赶在攻击到达目标系统之前将其检测出来并加以拦截。

NIDS 与基于网络的 IPS （NIPS）之间的明显差别在于 NIPS 是放在承载通信流的内联线路上的。

所有通信流都必须从 NIPS 经过，而 NIPS 可以选择转发哪些通信流，以及分析拦截哪些通信流。

主动 NIDS 没有放在内联线路上，只能在攻击到达目标之后把它查出来并拦截，不能预防攻击。

NIPS 可以使用基于知识检测或基于行为检测。

当前趋势是用 NIPS 取代 NIDS。

17.2.4 具体预防措施

1. 蜜罐/蜜网

蜜罐（honeypot）是为充当引诱入侵者或内部人员威胁的陷阱而创建的单个计算机。

蜜网（honeynet）是两个或多个连接在一起来冒充网络的蜜罐。

蜜罐的目的是引起入侵者的注意，使他们远离承载者重要资源的合法网络。

蜜罐使管理员能在不影响实时环境的情况下观察攻击者的活动。

可以作为抵御零日利用的有效手段。

蜜罐和蜜网可以放在网络任何地方，经常会放在虚拟系统中。

管理员经常给蜜罐配上伪缺陷（pseudo-flaw），以模拟众所周知的操作系统漏洞。

诱惑和诱捕的问题

诱惑是不需要蜜罐主任诱导进入的蜜罐。

诱捕是蜜罐主人诱导进入的蜜罐。

诱捕是非法的，需要关注法律。

2. 警示

警示告知用户和入侵者基本的安全方针策略。

任何在线活动都会被审计和监测。

警示的措辞很重要。

能以某种方式登录系统的未授权者也会看到警示。

警示既告知授权用户也告知未经授权使用者。

3. 反恶意软件

抵御恶意代码的最重要保护手段是使用带最新签名文件并具备启发式能力的反恶意软件程序。

定期更新签名（目前是自动一天检查几次更新）

多管齐下的方式拦截恶意软件和检测任何进入的恶意软件。

具备内容过滤能力的防火区（或专用内容过滤设备）部署在边界上。

专用反恶意软件安装在邮件服务器上，检测过滤电子邮件传递的恶意代码/

每个系统都安装检测和拦截恶意软件的反恶意软件程序。

每个系统安装反恶意软件程序在拦截互联网内容外，有助于使系统免受任何来源的病毒。

每个系统上只安装一种反恶意软件。

严格遵守最小特权原则，恶意程序无法安装或权限较低。

教育用户了解恶意代码的危险性、攻击者的手段以及如何限制风险。

4.白名单和黑名单

白名单：允许列表

黑名单：拦截列表/拒绝列表

用于控制哪些应用程序可以运行，哪些应用程序不能运行的方法之一。

针对应用程序使用允许列表和拒绝列表，有助于防止恶意软件感染。

一个系统只能使用一个列表：要么使用运行列表，要么使用拒绝列表。

有些允许列表标识了用哈希算法创建哈西函数的应用程序。

5.防火墙

防火墙属于预防和技术控制，力求通过技术手段来防止安全事件发生。

防火墙基本原则

  • 拦截路由器上的定向广播
  • 在边界拦截私网 IP 地址

基本防火墙根据 IP 地址、端口和一些带协议编号的协议过滤通信流。

防火墙放在网络边界，监视所有进出通信流。

防火墙执行 ACL 规则，允许特定通信流通过，以隐式拒绝规则结束。

防火墙阻止 ICMP 回声请求来拦截 ping 攻击。

所有 ICMP 通信流都使用编号为 1 的协议。

第二代防火墙增加了额外的过滤功能。

例如：
应用程序层面网关防火墙可根据特定应用程序的要求过滤通信流；

电路层面网关防火墙（circuit-level gateway firewall）根据通信电路过滤通信流。

第三代防火墙（状态检测防火墙（stateful inspection firewall）和动态包过滤防火墙）根据通信流在流量中的状态过滤通信流。

应用程序防火墙控制进出特定应用程序或服务的通信流。例如：WAF

下一代防火墙（NGFW）集多种过滤功能于一身，形成一种统一威胁（unified threat management）手段。

下一代防火墙包括包过滤、状态检查、包检查技术、黑白名单过滤，包含入侵检测和入侵预防功能。

6.沙箱

沙箱（sandboxing）为应用程序提供安全边界，可防止应用程序和其他应用程序交互。

反恶意软件应用借助沙箱技术测试未知应用程序。

沙箱技术可以防止该应用程序感染其他应用程序或操作系统。

许多反恶意软件程序供应商利用沙箱虚拟化技术观察恶意软件的行为。

7.第三方安全服务

必须向外部实体保证第三方服务提供者遵守具体安全要求。

PCI DSS 不允许组织将自己的责任外包出去。

17.3 日志记录和监测

日志记录和监测程序可帮助组织预防事件，并在有事件发生的时候做出有效响应。

日志记录机制把事件记录到各种日志中并对这些事件进行监测。

日志记录与监测结合起来使用，组织可跟踪、记录和审查活动，形成全面的问责体系。

17.3.1 日志记录技术

日志记录（logging）是将有关事件的信息写入日志文件或数据库的过程。

日志记录捕捉事件、变更、消息和其他描述系统上活动的数据。

日志会记录一些细节，比如何时、何地、何人、发生了何事以及如何发生的。

安全身份识别和验证流程作为问责先觉条件的要求。

审计不仅仅涉及日志记录，要审查或检查整个环境的合规情况。

1. 常见日志类型

安全日志

安全日志记录对文件、文件夹、打印机等资源的访问。

系统日志

系统日志记录系统事件，例如系统启动、关闭，服务启动、关闭，服务属性修改事件记录。

检测系统何时重启或检测服务何时关闭或修改的日志，可以帮助管理员发现潜在的恶意活动。

应用日志

应用日志记录有关具体应用程序的信息。

应用程序开发人员选择记录哪些内容。

防火墙日志

防火墙日志记录与到达防火墙的任何通信流相关的事件。

包括防火墙放行的通信流和拦截的通信流。

通常记录关键数据包信息，源、目的 IP 及端口。不记录数据包的实际内容。

代理日志

代理服务器可以提高用户访问互联网的效率，能控制允许用户访问哪些网站。

代理日志具备记录详细信息的能力。

记录访问了哪些网络以及在网站逗留了多长时间

记录用户何时试图访问已被禁止访问的站点

变更日志

变更日志记录系统的变更请求、批准和实际变更，是变更总流程中的一部分。

变更日志可以由人工创建、也可以由内部网页创建。

变更日志可以用于追踪得到批准的变更。

变更日志可以为灾难恢复计划提供帮助。

日志记录通常是操作系统以及大多数应用程序和服务本身就就具备的一个性能。

特权账号发生的事件应该被纳入日志计划。

17.5 本章小结

17.6 考试要点

列出和描述事件管理步骤

事件管理步骤：检测、响应、抑制、报告、恢复、补救和总结教训。

检测并证明有事件发生后，第一响应是限制或控制事件的范围，同时保护证据。

根据相关法律，组织可能需要把事件上报相关部门。

如果个人身份信息（PII）受到影响，则还需要把情况通知相关个人。

补救和总结教训阶段包括进行根本原因分析，以确定原因和提出解决方案，以防事件再次发生。

了解基本预防措施

基本预防措施可以防止许多事件发生。

这些措施包括保持系统即时更新、移除或禁用不需要的协议和服务、使用入侵检测和预防系统、使用配备了最新签名的反恶意软件程序以及启用基于主机和网络的防火墙。

了解白名单和黑名单的差异

软件白名单提供一个得到批准的软件列表，以防止未被列入名单的任何其他软件被安装到系统中。

黑名单则提供一个未得到批准的软件列表，以防止被列入名单的任何软件被安装到系统中。

了解沙箱

沙箱提供了一个隔离的环境，可以组织沙箱内运行的代码与沙箱外的元素交互。

了解第三方提供的安全服务

第三方安全服务可帮助组织增强内部员工提供的安全服务。

许多组织用基于云的解决方案来增强内部安全。

了解僵尸网、僵尸网控制者和僵尸牧人

僵尸网可以调动大量计算机发动攻击而形成一种重大威胁，因此，有必要了解什么是僵尸网。

僵尸网是遭入侵的计算设备（通常被称为傀儡或僵尸）的集合体，形成一个网络，由被称为僵尸牧人的犯罪分子操控。

僵尸牧人通过 C&C 服务器远程控制僵尸，经常利用僵尸网对其他系统发起攻击，或发送垃圾邮件或网络钓鱼邮件。

僵尸牧人还把僵尸网的访问权出租给其他犯罪分子。

了解拒绝服务（DoS）攻击

DoS 攻击阻止系统响应合法服务请求。

破坏 TCP 三次握手的 SYN 洪水攻击是一种常见的 DoS 攻击手段。

即便较老式的攻击由于基本预防措施的拦截而在今天已不太常见，依然会遇到这方面的考题，因为许多新式攻击手段往往只是旧方法的变体。

smurf 攻击利用一个放大网向受害者发送大量响应包。

死亡之 ping 攻击向受害者发送大量超大 ping 包，导致受害者系统冻结、崩溃或重启。

了解零日利用

零日利用是指利用一个除攻击者以外其他任何人都不知道或只有有限的几个人知道的漏洞的攻击。

从表面上看，这像是一种无从防范的未知漏洞，但基本安全保护措施还是能对预防零日利用提供很大帮助的。

通过移除或禁用不需要的协议和服务，可以缩小系统的受攻击面

通过启用防火墙，能封锁许多访问点

通过采用入侵检测和预防系统，可以帮助检测和拦截潜在的攻击

通过使用蜜罐等工具，可以帮助保护活跃的网络

了解中间人攻击

当一名恶意用户能够在通信线路的两个端点之间占据一个逻辑位置时，便意味着发生了中间人攻击。

尽管为了完成一次中间人攻击，攻击者需要做相当多的复杂事情，但他从攻击中获得的数据量也是相当大的。

了解入侵检测和入侵预防

IDS 和 IPS 是抵御攻击的重要检测和预防手段。

需要了解基于知识的检测（使用了一个与反恶意软件签名库类似的数据库）和基于行为的检测之间的区别。

基于行为的检测先创建一条基线以识别正常行为，然后把各种活动与基线相比较，从而检测出异常活动。

如果网络发生改动，基线可能会过时，因此环境一旦发生变化，基线必须马上更新。

认识 IDS/IPS 响应

IDS 可通过日志记录和发送通知来被动做出响应，也可通过更改环境来主动做出响应。

有人把主动 IDS 称为 IPS 。

IPS 被放置在承载通信流的内联线路上，可在恶意通信流到达目标之前拦截它们。

了解 HIDS 与 NIDS 的区别

基于主机的 IDS（HIDS）只能监测单个系统上的活动。缺点是攻击者可以发现并禁用它们。

基于网络的 IDS（NIDS）可以监测一个网络上的活动，而且是攻击者不可见的。

描述蜜罐和蜜网

蜜罐是通常用伪缺陷和假数据来引诱入侵者的一个系统。

蜜网是一个网络里的两个或多个蜜罐。

管理员可在攻击者进入蜜罐后观察他们的活动，攻击者只要在蜜罐里，他们就不会在活跃网络中。

了解拦截恶意代码的方法

将几种工具结合起来使用时可拦截恶意代码。

反恶意软件程序安装在每个系统、网络边界和电子邮件服务器上，配有最新定义，是最明显的工具。

基于最小特权原则等基础安全原则的策略也会阻止普通用户按照潜在的恶意软件。

针对风险和攻击者常用的传播病毒的方法对用户开展教育，也可帮助用户了解和规避危险行为。

了解日志文件的类型

日志数据被记录在数据库和各类日志文件里。

常见的日志文件包括安全日志、系统日志、应用日志、防火墙日志、代理日志和变更日志。

日志文件应该集中存储，以限制访问权限等方式予以保护，而归档日志应设置为只读，以防有人篡改。

了解监测以及监测工具的用途

监测是侧重于主动审查日志文件数据的一种审计形式。

监测用于使行事主体对自己的行为负责以及监测异常或恶意活动。

监测还用于监控系统性能。

IDS、SIEM 等监测工具可以自动持续进行监测并提供对事件的实时分析，包括监测网络内的情况、进入网络的通信流和离开网络的通信流（也叫出口监测）。

日志管理包括分析日志和归档日志。

解释审计踪迹

审计踪迹是在将有关事件及事发情况的信息写入一个或多个数据库或日志文件中时创建的记录。

审计踪迹可用于重建事件、提取事件信息、证明罪责或反驳指控。

使用审计踪迹是执行检测性安全控制的一种被动形式。

审计踪迹还是起诉犯罪份子的基本证据。

了解如何保持问责

通过使用审计，可保持对个人行事主体的问责。

日志记录用户活动，用户要对记录在案的行为负责。这对用户形式良好行为习惯、遵守组织安全策略有着直接的促进作用。

了解抽样和剪切

抽样也叫数据提取，是指从大量数据中提取特定元素，构成有意义的概述或摘要的过程。

统计抽样利用精确的数学函数从大量数据中提取有意义的信息。
