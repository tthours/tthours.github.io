error_fork_cannot_allocate_memory

背景

CentOS6中终端中执行命令提示报错

报错原因

系统进程数不足导致

解决方案

排查过程

查看Linux最大进程数配置

sysctl kernel.pid_max

查看当前Linux进程数

ps -eLf | wc -l

具体解决方案

进程数即将到设置的上线，修改最大进程数配置

临时生效

echo 1000000 > /proc/sys/kernel/pid_max

长久生效

echo “kernel.pid_max=1000000” >> /etc/sysctl.conf

sysctl -p
