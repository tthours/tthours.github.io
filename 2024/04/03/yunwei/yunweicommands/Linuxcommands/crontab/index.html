crontab

简介

crontab(定时任务)，是系统为用户提供定时执行任务的功能。

语法

不同版本的系统，都会在通用语法的之后再扩展一些语法

crontab命令语法：

crontab [-u user] [-l | -r | -e] [-i] [-s]

-u user：指定用户，不指定表示当前用户，只有管理员才可以指定用户，普通用户使用-u会出现如下提示

-l：list的意思，查看定时任务列表

-e：edit的意思，编辑指定定时任务，退出编辑状态之后会自动重新加载（reload）定时任务

-r：remove的意思，删除定时任务，如果有权限就会直接删除

-i：和-r搭配使用，会出现提示是否删除，输入y/Y就会删除

centos7扩展语法，增加了对集群的支持，-c/-n host

centos8扩展语法，增加了对集群的支持，-c/-n host

-V：显示版本

crontab格式

基本格式

* * * * * command(要运行的命令)

# +—————- minute 分钟(0 - 59)

# | +————- hour 小时(0 - 23)

# | | +———- day 日期(1 - 31)

# | | | +——- month 月份(1 - 12)

# | | | | +—- week 星期(0 - 7) (星期天=0 or 7)

# | | | | | # * * * * * 要运行的命令**

特殊字符

* 表示允许范围内任意数字

/ 表示每单元，*/5:每5个单元

a-b 表示从a-b的数字

a,b,c 表示在a,b,c的时间点执行

示例:

不同级别crontab

用户crontab

/var/spool/cron目录

目录权限：

​ drwx——,也就是700，所属组为root:root,表示只有root有操作权限

文件权限：

​ -rw——-，也就是600，所属组为对应用户，表示只有对应对应用户有查看编辑权限（root,有所有权限）

主系统crontab

/etc/crontab

文件权限为 -rw-r–r–，也就是644，表示任何用户都可以查看，但只有 root 可修改

周期性执行脚本目录

/etc/cron.hourly:每小时执行

/etc/cron.daily:每天执行

/etc/cron.weekly:每周执行

/etc/cron.monthly:每月执行

目录权限均为drwxr-xr-x，也就是766

定时任务调用流程

centos7

ll /etc/cron.d

cat /etc/cron.d/0hourly

ll /etc/cron.hourly/

cat /etc/cron.hourly/0anacron

cat /etc/anacrontab

cat /etc/anacrontab

注意事项

crontab -l 无法查看到 /etc/cron.daily 目录下的定时任务
