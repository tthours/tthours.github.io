error_mac_更换网络环境DNS无法切换

背景

Apple M1 macOS Sonoma 14.5

mac电脑从 A 网络环境切换到 B 网络环境后，出现可以 ping 通，但是无法解析域名的情况

排查过程

基础配置

1、检查DNS配置文件 /etc/resolv.conf 是 A 网络环境的DNS

2、同时使用 scutil –dns 查看本机 DNS配置

3、系统界面查看当前网络DNS配置

设置-网络- Wi-Fi - 详细信息 - DNS ，是设置的B网络DNS

尝试解决方案1

修改resolv.conf配置文件，配置没生效，依旧无法解析域名

1、终端修改DNS配置为B网络DNS

10.211.1.8 和 10.211.133.8 修改为 223.5.5.5

2、刷新DNS缓存

sudo dscacheutil -flushcache

sudo killall -HUP mDNSResponder

3、重新检查 resolv.conf 文件，确认已修改为 B 网络环境DNS

4、scutil –dns 查看本机 DNS配置，发现没有生效，还是之前的配置

尝试解决方案2

小伙伴提醒，不需要手动配置DNS网关，运营商会默认指定

1、界面删除DNS

删除233.5.5.5，默认是本地网关

新开终端，重新尝试ping www.baidu.com，依旧未成功
