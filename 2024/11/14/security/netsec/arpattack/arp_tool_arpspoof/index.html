arp_tool_arpspoof

简介

kali 默认会存在的 arp 攻击软件

安装

apt install dsniff

使用

查看版本和帮助

arpspoof -h

语法

arpspoof [-i interface] [-c own|host|both] [-t target] [-r] host

参数解读

-i interface 指定发送数据报的网卡

-c own|host|both 指定范围自己|特定主机|两者

-t target 指定攻击目标，如不指定则为 Lan上的所有主机

host 要拦截数据报的主机（也就是到那个机器的数据报），常为网关地址

-r 针对主机和目标主机，捕获双向流量（仅对 -t 生效）

验证

攻击机 kali 网卡信息

ip地址 192.168.3.162

mac地址 00:0c:29:ac:f2:b5

靶机Windows信息

ip地址 192.168.3.52

mac地址 00-0C-29-03-D3-D4

靶机 ARP 信息

网关ip地址 192.168.3.1

网关mac地址 04-f1-69-d3-a0-c4

攻击机上执行攻击命令

sudo arpspoof -i eth0 -t 192.168.3.52 -r 192.168.3.1

eth0 攻击机网卡

192.168.3.52 靶机IP地址

192.168.3.1 网关地址

查看靶机 ARP 信息

网关 192.168.3.1 mac地址已由 04-f1-69-d3-a0-c4 修改为攻击机 mac 地址00-0c-29-ac-f2-b5

结果

实现 arp 欺骗，靶机的数据都会传递到攻击机

现象：

靶机无法访问网络，攻击机开启端口转发功能后靶机可以正常访问网络

解决方案：

开启端口转发命令（默认值为 0 ，也就是不开启端口转发），该端口转发为临时设置，重启会失效，恢复为 0

注意

1、适用攻击机和靶机在同一网络下

2、攻击命令需要持续运行，停止运行之后，靶机网关 mac 地址会自动刷新
